| `expo.version` | `1.0.0` (left untouched — operator was unsure whether to bump) |
| `android.package` (== applicationId) | `com.vishnu.guardapp` |
| `android.versionCode` (in app.json) | bumped 6 → 7 by this build |
| `android.versionCode` actually shipped in APK | **10** (see §3) |

Discrepancy noted: with `appVersionSource: "remote"` in `eas.json`,
EAS pulls `versionCode` from its server and ignores the value in
`app.json`. EAS auto-incremented from 9 (last server-side value) to
**10**. The app.json bump 6 → 7 has no effect on this build and should
be reconciled (either delete the local field per the EAS warning, or
flip `appVersionSource` to `local`). For this build the practical
outcome — versionCode is monotonically larger than the previous build —
is satisfied either way.

EAS warning (verbatim):
```
android.versionCode field in app config is ignored when version source
is set to remote, but this value will still be in the manifest available
via expo-constants. It's recommended to remove this value from app config.
```

## 11. Rebuild — 2026-04-24 (Bug 1 + Bug 2 fixes)

**Trigger** — operator reported two on-device errors after the §6 install:
1. **Upload Failed: "Cannot convert undefined value to object"** — surfaced
   when the guard tapped the camera-upload path. Root cause: prod API
   was still on audit-base `e2fec53` (returning the pre-D1 PUT-presigned
   `{ url }` shape) while the mobile shipped with the D1 multipart-POST
   shape. The mobile-side `Object.entries(presign.fields)` then ran
   against `undefined` and threw the cryptic TypeError.
2. **Enhancement Failed: "404 model: claude-sonnet-4-20250514"** — the
   `/api/ai/enhance-description` endpoint hit the retired Sonnet 4 model
   (Anthropic retired it 2026-04-20).

### 11.1 Code changes shipped in this rebuild

| File | Change | Resolves |
| --- | --- | --- |
| `apps/mobile/lib/uploadToS3.ts` | Defensive shape-validation guard added before `Object.entries(presign.fields)`. Throws an actionable error ("Upload service returned an unexpected response shape. The API may need to be redeployed…") instead of TypeError. | Bug 1 (mobile-side debuggability) |
| `apps/api/src/routes/ai.ts` | Extracted `ANTHROPIC_MODEL` env var, default `claude-sonnet-4-5-20250929` (was hardcoded retired model). Server-side only — **does not ship in APK**. | Bug 2 (server-side, requires Railway redeploy) |
| `apps/api/scripts/test-upload-flow-mobile.ts` | New regression test — 36 assertions covering the API/mobile contract across all 3 upload contexts + all defensive failure modes. Catches future deployment skew at CI time. | Regression prevention |

Test result: `npx tsx apps/api/scripts/test-upload-flow-mobile.ts` → **36/36 PASS** before rebuild kicked off.
